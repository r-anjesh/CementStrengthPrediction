How It Works - Cement Strength Prediction

1. Problem Statement

The goal of the Cement Strength Prediction system is to build a regression model that can predict the compressive strength of concrete based on various features. These features are derived from training data, including components like cement, blast furnace slag, water-cement ratio, and more. By training a machine learning model with this data, we can predict how different mix ratios and conditions affect the strength of the cement.

2. Data Description

The dataset used for training contains various concrete components and the age of the mix, along with the resulting compressive strength of the concrete (in MPa). Below is an overview of the variables involved:

Name Data Type Measurement Description
Cement Quantitative kg in a m3 mixture Input Variable
Blast Furnace Slag Quantitative kg in a m3 mixture Input Variable
Fly Ash Quantitative kg in a m3 mixture Input Variable
Water Quantitative kg in a m3 mixture Input Variable
Superplasticizer Quantitative kg in a m3 mixture Input Variable
Coarse Aggregate Quantitative kg in a m3 mixture Input Variable
Fine Aggregate Quantitative kg in a m3 mixture Input Variable
Age Quantitative Days (1-365) Input Variable
Concrete Compressive Strength Quantitative MPa Output Variable

3. Data Validation

Before using the data for model training, we validate the provided files using a variety of checks:

  • File name validation based on a regex pattern
  • Validation of the number of columns and column names
  • Checking for correct data types and handling missing values
  • Disqualifying any files with invalid or NULL data

4. Model Training

After validating the data, multiple machine learning models are applied to train the prediction system. The process includes:

  1. Data Export: The validated data is exported as a CSV file.
  2. Data Preprocessing: This involves handling missing values, log transformation of features, and scaling the data.
  3. Clustering: We apply KMeans clustering to group the data into clusters, allowing us to handle different data patterns more effectively.
  4. Model Selection & Tuning: We test and compare various machine learning models (e.g., Random Forest, Gradient Boosting, Linear Regression, etc.) for each cluster. After selecting the best-performing model, we fine-tune its hyperparameters for improved accuracy and reliability.

5. Prediction Process

To make predictions, the system follows this process:

  1. Enter the absolute file path for the data or choose a default file.
  2. For custom predictions, you can upload your own CSV file.
  3. The model processes the input data and generates predictions.
  4. Results are outputted to a CSV file saved at Prediction_Output_File/Predictions.csv.
